OSSIM Threat Intelligence

First thing for Threat Intelligence for the OSSIM version, is to check that your Alienvault is using Suricata plugin instead of Snort.

*To do this you will need to click Configuration -> Deployment, from the homepage. Once the Deployment model is loaded, double click the only Alienvault in the list

*Click Sensor COnfiguration, then change the view to Detection, check its on.

You can also turn on a couple other features that will help you monitor your assets better. Those two features are KISMET and NETFLOW.

To turn these two on you will need to go to configuration > Deployment and select the sensors option. Select your first sensor and click modify.

From there you will be presented with options for your sensor, scroll down to the check boxes and check KISMET, as well as go to the bottom of the page for the NETFLOW options and select Run and Configure.

Now your VM is ready to monitor incoming threats, but we need to check our alarms to make sure you have the right ones set up correctly.

To do this head to Threat Intelligence under the Configuration drop down. Select Data source from the top bar on that page. That page will load a lot of different sources to raise alarms.

Select and of the symbol that looks like a Page with the magnifying class (the last colomn in the row) this will load the editing source page.

To raise specific alarms for what you are trying to detect, you will need to raise the priority(alarm importance) and reliability(how much you trust this is a true attack) to capture alarms for you specific events.

So now we can start sending attacks through Armitage (a Cyber Security Class tool for Red and Blue teaming)

Once you have got in your kali linux VM and started sending hail mary attacks, the alarms are already pre-configured to be set when these attacks occur.

After you have gained access to the Windows XP machine through Armitage, come back to alienvault and we will check our Secuity Events and Alarms to check the details were captured correctly.

Directives

In order to see alarms for attacks you will need to set up a directive to alarm for that specific attack.

To set up a directive go to Configuration -> Threat Intelligence and select directives, when that page is loaded select new Directive.

These next steps will show you how to set up a DDOS attack for alarms, but any attack can be alarmed by repeating these steps and changing what is being detected.

Name the directive DDOS - Alert Test, Inten is Exploitation and Installation, Strategy will be Denial of Service - Hacking Tool.

Set priority to 5 and hit next. Name the Rule DDOS - Test Alert and Select Next.

Change event types to Taxonomy on the next page and add Alarm, as well as Intrusion Detection to the list on the left. On the next screen don't change anything and hit next. Same thing on the Network page. Select 10 for Reliability and select Finish

Now your Directive has been set up and you will have to reset the Alienvault server for the changes to take place.
